# Dauction — KYC service

Identity-verification bounded context for Dauction. Owns document references
(Emirates-ID / passport), GCC phone OTP verification, and the admin approval
queue. A built from the `mequq/go-template` clean-architecture template (Go
module `application`). It owns its own Postgres database (`kyc`) and never reads
another service's tables; cross-service integration is via NATS/JetStream events.

## Role

- Accept a KYC submission: document type + document **reference** (never a blob)
  + a GCC E.164 phone, and issue a 6-digit OTP challenge (only a SHA-256 hash of
  the code is stored; in non-production the code is logged/returned for dev).
- Verify the OTP, moving the submission into the admin queue.
- Let the House approve / reject a queued submission with a machine reason code.
- Emit `kyc.approved` / `kyc.rejected` via a transactional outbox so a decision
  and its event commit atomically; a background relay publishes to NATS.

## State machine

```
STARTED ──verify OTP──▶ SUBMITTED ──approve──▶ APPROVED
   │                        │
   └─ (OTP issued)          └──reject──▶ REJECTED ──(resubmit → new STARTED row)
```

Illegal transitions return `biz.ErrResourceInvalid`. A REJECTED submission is
terminal for that row; the account may resubmit, which inserts a fresh row. An
APPROVED or in-flight submission blocks a new `start`.

## Routes (mounted under `/apis` by the gateway)

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/apis/kyc/start` | Begin KYC; issue an OTP challenge |
| `POST` | `/apis/kyc/verify` | Verify OTP → submission enters the admin queue |
| `GET`  | `/apis/kyc/status` | Caller's current KYC state |
| `GET`  | `/apis/admin/kyc` | Admin: pending queue (SUBMITTED) |
| `POST` | `/apis/admin/kyc/{id}/approve` | Admin: approve → emits `kyc.approved` |
| `POST` | `/apis/admin/kyc/{id}/reject` | Admin: reject (reason code) → emits `kyc.rejected` |

The caller identity is taken from the gateway-set `X-Account-Id` header. Error
responses are language-neutral machine codes (`RESOURCE_NOT_FOUND`,
`RESOURCE_EXISTS`, `RESOURCE_INVALID`, `ACCESS_DENIED`) — the React client maps
them to localized copy.

## Events

| Subject | Payload (proto3 JSON, `dauction.events.v1`) |
|---|---|
| `kyc.approved` | `KycApproved { account_id, submission_id }` |
| `kyc.rejected` | `KycRejected { account_id, submission_id, reason }` (reason is an `ErrorCode` value-name) |

Both are wrapped in the shared `EventEnvelope` (eventId, idempotencyKey,
producer=`kyc`, occurredAt, type, version). The idempotency key is stable per
(submission, decision) so re-publishes dedup downstream.

## Tables (own DB `kyc`)

- `kyc_submission` — one verification attempt per row; state CHECK-constrained.
- `otp_challenge` — OTP per submission; only the code hash, attempts, expiry.
- `outbox` — transactional outbox (unique `idempotency_key`, `published_at`).

## Configuration

`config.yaml` (copy from `config.example.yaml`) is the runtime source of truth;
`APP_`-prefixed env vars overlay it (`_`→`.`). Key settings:

| Key | Env | Default | Notes |
|---|---|---|---|
| `server.http.addr` | `APP_SERVER_HTTP_ADDR` | `:8080` | HTTP listen |
| `datasource.postgres.dsn` | `APP_DATASOURCE_POSTGRES_DSN` | `…@pg-kyc:5432/kyc` | own DB |
| `datasource.nats.dsn` | `APP_DATASOURCE_NATS_DSN` | `nats://nats:4222` | event bus |
| `kyc.otpTtlSeconds` | `APP_KYC_OTPTTLSECONDS` | `300` | OTP lifetime |
| `kyc.maxAttempts` | `APP_KYC_MAXATTEMPTS` | `5` | OTP attempt cap |
| `kyc.production` | `APP_KYC_PRODUCTION` | `false` | suppress dev OTP exposure |
| `collector.exporters.otlp.endpoint` | — | `jaeger:4317` | OTLP traces/metrics/logs |

## Quickstart

```sh
cp config.example.yaml config.yaml
make generate                      # regenerate Wire DI graph + go.mod tidy
go run ./cmd/app --config ./config.yaml
```

## Architecture

Clean architecture, dependency direction
`cmd → app → service/handler → biz → repo → datasource → entity`, wired with
Google Wire. Vertical slice per resource (`kyc`):
`entity → biz (Usecase/Repository ifaces + impl) → repo (raw pgx $1 SQL + OTel)
→ dto (validate tags + mappers) → handler (implements service.Handler, swag)`.
Mocks for `internal/biz` are generated by mockery into `internal/mocks`.
